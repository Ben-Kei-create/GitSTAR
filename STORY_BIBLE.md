# GitSTAR — Story Bible

> *「すべての記録が失われた世界で、たった一つのコマンドから歴史を紡ぎ直す。」*

このドキュメントは GitSTAR の世界観・キャラクター・物語の根幹を定義する正典である。
すべての台詞・UI・演出はこの文書に従う。

---

## 第零章 — 世界の名 *Gitia*

**Gitia（ギティア）** とは、「記録」が物理法則として機能する世界である。

人々の存在、建物、記憶、感情、すべてが **アーカイブの木**（The Archive Tree）と呼ばれる
巨大なルートリポジトリにコミットされることで「実在」を保っていた。

> コミットされていないものは、存在しない。
> コミットされたものは、永遠に存在する。

これが Gitia の唯一にして絶対の摂理である。

---

## 第一章 — 大崩壊 *The Unwriting*

### 起源の災厄

37年前、世界に **腐敗（Corruption）** が忍び寄った。

ある夜、誰かがアーカイブの木に書き込んだ1つの**不完全なコミット**が、
参照を循環させ、世界の時間そのものを無限ループさせ始めた。

修正すれば別の場所が壊れる。
merge すれば感染が広がる。
あらゆる手段が通用しなかった。

### マスターの選択

世界を救う唯一の方法は、**腐敗が始まる前まで世界を巻き戻すこと**だった。

最後の記録者 **マスター** は、世界のルートに対して、
震える指で禁忌のコマンドを打ち込んだ。

```bash
git reset --hard <腐敗前のハッシュ>
```

この瞬間、**大崩壊（The Unwriting）** が起きた。

腐敗は消えた。
そして同時に、世界のすべての記憶・人々・歴史も消えた。

マスターは、愛する者たちの存在ごと、世界をリセットしたのだ。

---

## 第二章 — 現在の Gitia *37年後*

### 消えゆく世界

リセットされた世界は、もう一度コミットされ直さなければ実在を保てない。
しかしマスター以外に記録者はおらず、世界は少しずつ「存在しなかったこと」になり続けている。

家の輪郭がぼやけ、人の顔が思い出せなくなり、
言葉が、名前が、空の色が、ゆっくりと薄れていく。

### 終焉のタイマー — *git gc*

そして今、世界のシステムが最後の処理を始めようとしている。

**大掃除（Garbage Collection / `git gc`）**

参照されていないオブジェクト——
誰のコミットからも辿れない、宙に浮いたデータ——
それらを **完全に・不可逆に・永久に消去する**処理。

現在の Gitia の人々は、ほぼ全員が「Dangling（参照なし）」状態である。
gc が完了したとき、世界は本当に終わる。

> 主人公がこの世界に呼ばれたのは、
> gc のタイマーが鳴り始めた、まさにその瞬間だった。

## 第三章 — 世界の物理法則

Gitia における Git コマンドは、単なる操作ではない。**世界そのものに干渉する儀式**である。

| コマンド | 物語上の意味 |
|---|---|
| `git init` | 失われた領域に**新しい記録の火**を灯す。世界に小さな実在の核を作る。 |
| `git add` | 曖昧な存在を**「記録対象」として神に申告する**。 |
| `git commit` | 対象に **40文字の魂のID（ハッシュ）** を刻み、世界に永遠にピン留めする。 |
| `git push` | 自分が紡いだ歴史を、**他の生存者の世界線と共有**する。 |
| `git pull` | 他の世界線から、**自分の知らなかった真実**を受け取る。 |
| `git branch` | **並行する歴史**を生み出す。Aの世界とBの世界の同時存在。 |
| `git merge` | 2つの世界線を**1つの真実に統合**する。 |
| `git diff` | **2つの実在の差分**を見抜く。 |
| `git stash` | 未確定の存在を**異次元に一時退避**する。 |
| `git revert` | 過去の傷を**消さずに、打ち消す決意**を新しい歴史として刻む。 |
| `git reset` | **歴史そのものを巻き戻す**禁忌の力。`--hard` は世界を消す。 |
| `git reflog` | **神の記憶**。すべての操作が刻まれた、人の世から見えない深淵の記録。 |
| `git rebase` | 歴史を**美しく書き直す**儀式。整えることで未来を守る。 |
| `git cherry-pick` | 別の世界線から、**特定の出来事だけを抽出**する。 |
| `git tag` | コミットに**永遠の名前**を与える。「あの瞬間」を語り継ぐための錨。 |
| `git blame` | コードの行ごとに**「誰が、いつ、何のために」**書いたかを問う鏡。 |
| `git bisect` | 失敗の根源を**二分探索の灯**で照らす。 |
| `.gitignore` | アーカイブの木に**残してはいけないもの**を定める律法。 |

### 中心概念 — *ハッシュは魂*

世界に流通するハッシュ値（`a3f8b2c...`）は、**Gitia における唯一の不変要素**である。

```
人は忘れる。
建物は崩れる。
記憶は薄れる。

しかしハッシュだけは、永遠に変わらない。
```

主人公がコミットを打つたび、消えかけている誰かに **40文字の魂のID** が刻まれ、
その人物は世界に繋ぎ止められる。

`git log` の画面に流れるハッシュ列は、Gitia における **「死者の名簿」であり、生者の証明書**である。

## 第四章 — キャラクター

### 🌟 コミ — *.git に宿る記憶の精霊*

**正体**：大崩壊の時に飛び散った記録の欠片が、自我を持った存在。

**設定**：
- 物語開始時は半透明・声がノイズ交じり・輪郭が崩れている
- 主人公がコミットを重ねるたびに、解像度が上がっていく
- 最終的には完全に実体化する

**役割**：プレイヤーの最も近い相棒。優しい、ちょっと頼りない。

**解像度の進化**：

| 時点 | 状態 |
|---|---|
| Arc 1 開始 | 半透明、ノイズ、声が時々途切れる |
| Arc 1 終了 | 輪郭がはっきりしてくる |
| Arc 2 | 色彩が完全に戻る |
| Arc 3 コンフリクト | 一瞬また透ける（不安定化） |
| Arc 4 reflog | ピンチ → 救出で完全実体化 |
| Arc 6 終局 | 「ぼく、ちゃんと "いた" ね」 |

---

### 📜 マスター — *最後の記録者にして、世界を壊した男*

**正体**：37年前、腐敗を止めるために `git reset --hard` を打った張本人。

**設定**：
- 世界を壊した罪悪感を背負って生きている
- 次の世代に希望を託すことだけが、彼の贖罪
- 寡黙で重厚、だが優しい
- 自分の過去を直接語ることはほとんどない

**役割**：プレイヤーの師。重要な技を授ける。
時折見せる遠い目が、彼の罪と覚悟を物語る。

**真実の開示タイミング**：Arc 6 にて、すべてが明かされる。

---

### 🌿 ブラン — *分岐の民の末裔*

**正体**：大崩壊の瞬間、別のブランチに逃げ延びた一族の生き残り。

**設定**：
- 並行する複数の歴史を旅してきた
- 複数の「もしも」の世界を見てきた目
- 軽口を叩くが、その裏に喪失感
- 「main じゃない世界」の重要性を知っている

**役割**：Arc 2 で登場。チームプレイの導入役。
Arc 4 で gc により消えかかる。reflog で救出される。

---

### ⚡ コンフリックス — *矛盾の擬人化*

**正体**：2つの真実が衝突する時、世界に立ち現れる存在。

**設定**：
- 敵ではない。**判定者**である。
- 「Aが生き残った歴史」と「Bが生き残った歴史」が両立できない時に現れ、選択を迫る
- 巨大で、感情を持たない、しかし無慈悲ではない
- どちらの真実にも哀しみを感じている

**役割**：Arc 3 のメインボス的存在。
ただし倒すのではなく、**「どちらかを選び、もう片方を悼む」**ことで対話を終える。

---

### 🪶 リベイ — *時間の書き換え師*

**正体**：歴史を「整える」ことに執念を燃やす孤高の職人。

**設定**：
- かつてはマスターと共に世界を救おうとした
- 腐敗を「整えれば消せる」と信じていたが、できなかった
- 今は山奥に隠れ、後進に rebase の技術を伝える
- 完璧主義、寡黙、しかし弟子には優しい

**役割**：Arc 5 で登場する師匠。
歴史は「ただ残すだけでなく、美しく整えるべき」という哲学を伝える。

---

### 🌳 ログバア — *記録の番人、最長老*

**正体**：大崩壊以前の記憶を唯一保持している老人。

**設定**：
- 彼の目に映るものは全員、`git log` の断片として見える
- 「あんたのハッシュは a3f8 で始まるね」と挨拶する変人
- ただ、その目は決して間違わない
- マスターと旧知の仲。互いに何も語らない

**役割**：Arc 6 の真相開示の鍵を握る。

---

### 👻 ダングリングたち — *哀しき残滓*

**正体**：誰からも参照されず、消える運命にある「失われた歴史」の擬人化。

**設定**：
- **敵ではない**
- 主人公にすがりついてくる：「ぼくも、わたしも、歴史に残してほしい」
- 触れるとハッシュの欠片が見える（4文字程度しか残っていない）
- gc が発動すると、消える

**役割**：道中で出会う、世界の悲しみの象徴。
プレイヤーは時折、彼らをコミットに含めて救うことができる。
（救えないダングリングもいる。それも物語）

## 第五章 — Arc 構成 *6章 + アナザーストーリー*

各 Arc は**学ぶコマンド**ではなく、**プレイヤーが体験する感情**で定義される。

---

### Arc 1：ひとり開発編 — *火を灯す者*

**感情の弧**：絶望 → 小さな希望

**舞台**：廃墟の村の外れ。荒れ果てた小屋。

**物語**：
主人公は、何もない場所で目を覚ます。
半透明のコミ、寡黙なマスターと出会う。
「`git init` を打て」と言われ、震える手で実行する。
小さな火（.git）が灯る。コミの輪郭が、ほんの少しはっきりする。

**山場**：初めての `git commit`。
名もなき石ころに「最初のハッシュ」を刻む。
石ころが、確かにそこに「在る」状態になる。

**学ぶコマンド**：init, status, add, commit, log, push, pull, remote add

---

### Arc 2：チーム入門編 — *並行世界の合流*

**感情の弧**：孤独 → 仲間との出会い

**舞台**：別の世界線から流れ着いた **ブラン** との合流地点。

**物語**：
主人公は別のブランチを探索することで、**ブラン**と出会う。
ブランは、別の歴史線を旅してきた話を聞かせる。
2人が別々のブランチで世界を修復し、最後に `merge` で1つの歴史にする。

**山場**：初めての `merge`。2つの世界線が1つになる時、画面の星が増える。

**学ぶコマンド**：branch, checkout, merge, （diff, stash の予習）

---

### Arc 3：チーム実戦編 — *コンフリクトの悲劇*

**感情の弧**：信頼 → 衝突 → 痛みを伴う和解

**舞台**：2人で修復していた村に、**コンフリックス** が現れる。

**物語**：
ブランと主人公が、別々のブランチで「壊れた村」を修復していた。
合流（merge）しようとした瞬間、コンフリックスが現れる。
**「2人が修復した『村の形』が、両立できない」**

プレイヤーは選択を迫られる：
- ブランの修復案を採用する（自分の修復を捨てる）
- 自分の修復案を採用する（ブランの修復を捨てる）

どちらを選んでも、何かが失われる。

**山場**：コンフリクト解消の瞬間、画面の半分が暗くなる演出。
失われた歴史への黙祷。

**学ぶコマンド**：diff, stash, conflict resolve（add → commit）

---

### Arc 4：緊急対応編 — *黄泉返りの reflog*

**感情の弧**：恐怖 → 絶望 → 救済の閃光

**舞台**：深夜。世界の `git gc` が一部発動。

**物語**：

```
コミ：「あれ……ブランの声が……聞こえない」
マスター：「git gc が……始まったか」
[画面が崩れる演出。ブランの立ち絵がノイズに飲まれる]
```

`git log` を打っても、ブランのコミットが出てこない。
`git status` を打っても、痕跡がない。
ブランは、世界から「参照されていない状態」になり、消えかけている。

```
マスター：「これは……禁忌の技だ。
        だが今、それしか道はない。」
マスター：「git reflog ——"神の記憶" を覗け。」
```

プレイヤーが `git reflog` を実行。
画面に膨大なハッシュ列。その中に、ブランの最後のハッシュが点滅している。

```
マスター：「あった。あのハッシュを掴め！」
```

`git checkout <ブランのハッシュ>` で、ブランを引き戻す。

```
[ブランの立ち絵が、ノイズの中から少しずつ実体化]
ブラン：「……ぼく、消えかけてた？」
コミ：「うん。でも、ハッシュが、君を覚えてた。」
```

**山場**：reflog のスクロール演出。膨大なハッシュの中から1つを見つける緊張感。

**学ぶコマンド**：revert, reset (--soft / --mixed / --hard), reflog

> **本作の感情のピーク**

---

### Arc 5：熟練者編 — *歴史を整える者*

**感情の弧**：自信 → 謙虚さの獲得

**舞台**：山奥の隠れ家。**リベイ** との出会い。

**物語**：
ある程度のコマンドを使いこなすようになった主人公に、リベイが言う：

> 「君のコミット履歴は、汚い。
>  歴史は、ただ残すだけじゃダメだ。
>  美しく整えなければ、未来の誰かが読めない。」

リベイから rebase の技術を学ぶ。
`git rebase -i` で過去のコミットを統合・並び替え・修正する。
最後に、村の重要な瞬間に `git tag v1.0.0` を打ち、永遠の名前を与える。

**山場**：rebase -i で「散らかった歴史」が「物語」に変わる瞬間。
プレイヤーは「Git は記録するだけでなく、編集する道具でもある」と理解する。

**学ぶコマンド**：rebase, rebase -i, cherry-pick, tag

---

### Arc 6：チームリード編 — *新しい起点*

**感情の弧**：真実 → 受容 → 新たな決意

**舞台**：再建されつつある村。**ログバア**との対話を経て、すべてが明かされる。

**物語**：

ログバアが主人公にだけ告げる：
**「マスターのハッシュは、37年前で止まっとる。」**

主人公はマスターを問い詰める。
マスターは、ついに語る——大崩壊の真実、自分が引き金を引いたこと、
すべての記憶を消した代償と、その後の罪悪感を。

```
マスター：「私は、世界を救うために、世界を消した。
        その罪は、消せん。」
コミ：「……でも、マスターがいたから、ぼくは "いる"。」
```

プレイヤーは決断する：
**昔のアーカイブの木を完全復元する**のではなく、
**失われた歴史すべてを抱えたまま、新しい起点（git init）を宣言する**ことを。

最終ミッション：

```bash
git init  # 新しい世界の始まり
git add . # 失われた者たち、生き残った者たち、すべて
git commit -m "ここから、もう一度。"
git tag v∞
```

**山場**：最後のコミット。画面が、初めて完全な色彩を取り戻す。
コミが完全実体化し、初めて笑う：

> コミ：「ぼく、ちゃんと "いた" ね。」

**学ぶコマンド**：blame, bisect, log advanced, .gitignore, **git init（再演）**

## 第六章 — アナザーストーリー *腐敗の起源*

50コマンド以降のハードモード。

### コンセプト
本編クリア後に解放される **「腐敗の起源」** を辿る別軸ストーリー。
プレイヤーは時間を遡り、37年前の大崩壊の真因を調査する。

### 学ぶコマンド（候補）
- `git fetch` — 他の世界線を覗く
- `git show` — 過去の真実を直接見る
- `git clean` — 不要な痕跡の処理
- `git rebase --continue / --abort` — 過去の決断のやり直し
- `git config` — 自分自身のルールを定義
- `git submodule` — 入れ子の世界
- `git worktree` — 複数の現実を同時に持つ
- `git gc` — **自ら大掃除を発動する側に立つ覚悟**
- `git remote -v` / `git push --force-with-lease` — 他者との慎重な共有

### テーマ
> 「世界を壊したのは、本当にマスター一人だったのか？」

## 第七章 — ビジュアル方針

### レイヤー構成

```
Layer 0: 背景画像（差し替え可能 / Arc・状況ごとに変化）
Layer 1: ノイズ / フリッカー（世界の不安定さの表現）
Layer 2: Git フロー図（うっすら常時表示 — 世界を支える根）
Layer 3: キャラクター立ち絵（差し替え可能 / 解像度進化）
Layer 4: ダイアログUI / コマンド入力UI
```

### 進化する世界の演出

| 段階 | 背景 | フロー図 | キャラ |
|---|---|---|---|
| Arc 1 開始 | 色褪せ・廃墟 | 細い線・1点だけ | 半透明・ノイズ |
| Arc 2 | 少し色彩 | 線が増える | 輪郭明瞭 |
| Arc 3 | 嵐の予感 | 分岐が現れる | 安定 |
| Arc 4 危機 | 漆黒・点滅 | 線が震える | 一部消失 |
| Arc 5 | 落ち着いた青 | 線が整う | 安定 |
| Arc 6 終局 | 鮮やかな夜空 | 大樹のように太く光る | 完全実体化 |

### Git フロー図のゴースト表示

現在の `VerticalGitGraph` を全画面の背景レイヤーとして `opacity ≈ 0.06` で常時表示する。
プレイヤーがコミットを増やすたび、この「うっすら見える根」が太く、力強く光る。

> **世界を支えているのは、自分が打ったコマンドの軌跡だ。**
> プレイヤーがそれを「見える」形で常に感じられるように。

### 差し替え可能な設計

Swift の Protocol を使い、現在は仮実装、将来は差し替え可能に：

```swift
protocol CharacterArtwork {
    func makeView(resolution: Double) -> AnyView
}

protocol BackgroundArtwork {
    func makeView(timeOfDay: TimeOfDay, mood: Mood) -> AnyView
}
```

仮実装：SF Symbol + SwiftUI Shape でシルエット風に
本実装：`Image("komi_arc1")` などに差し替えるだけ

### コマンド入力UIの "ハッキング感"

カスタムキーボードの押下時に：
- 緑のフラッシュ
- 微弱な振動（haptic）
- ターミナル風のドット連打音（オプション）

「コマンドを打つ = 世界に介入する」という没入感を演出する。

## 設計上の不変原則

1. **Git の実装と物語の整合**：物語上の演出は、必ず実際の Git の挙動と一致させる。
2. **「学習」より「体験」**：コマンドの暗記ではなく、世界の物理法則の理解を促す。
3. **誰も完全な悪役ではない**：マスターも、コンフリックスも、ダングリングも、それぞれの正しさを持つ。
4. **希望は常に小さく始まる**：壮大な救済ではなく、1コミット、1ハッシュから。
5. **プレイヤーの操作が世界を変える**：すべての演出はプレイヤーのコマンド入力に応答する。
